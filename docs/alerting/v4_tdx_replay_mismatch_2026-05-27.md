## 5. 关键差异定位（已确认）

### 5.1 通达信回放覆盖范围不足早盘

`debug_replay_today_from_tdx()` 当前只拉 `MAX_HISTORY_BARS_1M=241` 根。

结果：日志 TRACE 从 `10:30` 左右开始，无法覆盖 `09:36` / `09:59`。

这意味着：
- 收盘后同源回放当前实现**无法验证早盘 BUY/SELL**是否应触发。
- 这不是信号逻辑本身漏报，而是回放窗口长度不足导致观测盲区。

### 5.2 11:27 BUY 在通达信同源数据下未过门槛

通达信 TRACE（11:27）显示：
- BUY: `setup=23`, `confirm=46`, `total=79`
- 因 `MIN_SETUP_SCORE=26`，被判定候选不足（未进入触发流程）。

而回测日志对应 11:27 BUY 能触发，说明：
- 该点位主要受数据源细节影响（setup 恰好差 3 分）；
- 属于硬门槛边界效应，不一定是代码 bug。

### 5.3 波段收紧开关对信号数影响显著

`WAVE_END_REQUIRE_EXCURSION=True` 明显减少信号（8 -> 5，akshare同日验证）。

因此实盘/回放/回测对比时，必须确保该开关一致，否则会误判“漏报”。

---

## 6. 当前最可能原因排序

1. **回放窗口不足**，导致早盘信号未被回放到（当前实现问题）。
2. **数据源差异 + setup硬门槛**，使 11:27 BUY 在通达信数据下不达标。
3. **波段收紧开关一致性**（True/False）可能影响是否出现额外 BUY。
4. 次要：同波段冷却在不同分钟路径下触发差异（如 13:29 SELL在回放被冷却挡掉）。

## 10. 最终结论

1. **漏信号根因**：回测曾用 akshare（前复权），实盘/回放用通达信；信号逻辑相同，数据源不同导致评分差异。
2. **通达信 1 分钟历史深度**（实测 `300959`）：约 **22800 根 / 95 个交易日**（例：`2025-12-30` ~ `2026-05-27`），可满足常规日内回测；更早区间需改用 `BACKTEST_DATA_SOURCE='akshare'` 或缩短区间。
3. **已改造**：回测默认走通达信，模块 `alerting/backtest_data_source.py`；配置项 `BACKTEST_DATA_SOURCE`（`tdx` | `akshare`）。绘图仍用同一 `df_1m` + `signals`，**不受影响**。
4. **对齐验证**（`300959`，`2026-05-27`，`WAVE_END_REQUIRE_EXCURSION=True`）：通达信回测 = 通达信回放 = **3 SELL / 0 BUY**（`09:59`、`13:28`、`14:28`）。此前 akshare 回测的 2 BUY 不应再作为实盘预期。
