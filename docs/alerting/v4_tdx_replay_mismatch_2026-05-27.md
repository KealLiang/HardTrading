## 13. V5 说明（2026-05-28）

### 13.1 与 V4 的使用关系

| 维度 | V4 | V5 |
|------|----|----|
| 继承关系 | 基类 `TMonitorBase` | **继承 V4**，复用取数、波段冷却、量能、涨停过滤、绘图 |
| 运行方式 | `MonitorManagerV4` + `__main__` | **相同**：改符号列表、回测区间、`BACKTEST_DATA_SOURCE` 即可 |
| 实盘/回测 | `IS_BACKTEST`、飞书推送、`watchlist.txt` | **相同** |
| 信号前缀 | `【V4信号】` | `【V5信号】`；RSI 语境异常时为 `**疑似BUY**` / `**疑似SELL**`（仅文案） |
| 产品定位 | 情绪局部衰竭拐点 | **相同**；不含横盘起爆/突破 |

结论：**使用上与 V4 基本一致，换文件即可**；差异在评分/入池/确认结构，不是另一套监控框架。

### 13.2 V4 / V5 逻辑对比

| 项目 | V4 | V5 |
|------|----|----|
| **入池** | `setup≥26` **且** `confirm≥26` | 路径 A：双 `≥24`；路径 B：`setup+confirm≥52` 且 `min≥20`（买卖对称） |
| **总分触发** | `total≥72` | 默认 `≥70`；或 `setup+confirm≥58` 且 `location≥6` 时 `≥68` |
| **setup** | 急杀/急拉（RSI 极值、触轨、20m 涨跌幅） | 同上 + **阴跌磨底 / 慢推升温**（窄幅磨底/磨顶，与急杀取较高分） |
| **confirm** | 偏单根（不再新低/高、RSI 连升、近 3 收盘突破等） | **结构型**：半窗低点抬高/高点降低、EMA 占比、RSI 斜率/背离、脱离极值；保留量能辅助 |
| **文案** | `RSI:当前` + setup/confirm 理由 | 同上；RSI 语境离谱时方向前加 **「疑似」**（不改变是否触发） |
| **RSI 指标** | 单条 `RSI(14)`，跨日连续 | **同 V4**（未改指标层） |
| **默认门槛松紧** | 更严 | 入池/总分略松；结构 confirm 有增有减 |

### 13.3 代码质量结论（审查 2026-05-28）

- **无** TRACE / debug 回放等临时代码；V5 通过 **继承 + 方法覆盖** 实现，非整文件复制粘贴。
- `triggered_signals[].type` 仍为 `BUY`/`SELL`（不含「疑似」），绘图与统计与 V4 一致；「疑似」仅出现在日志/飞书文案。
- 已知文案歧义：`setup` 里 `RSI极低/极高{x}` 来自 **EXTREME_WINDOW 内极值**，`RSI:{当前}` 为触发 bar；二者不一致时常标「疑似」。

## 11. 历史：V4 阶段优化目标（部分已由 V5 承接）

### 11.1 产品目标

- **用途**：盘中监控 **情绪拐点**，辅助盯盘，降低人工疲劳与情绪干扰。
- **不追求**：信号 100% 正确、覆盖每一个波动。
- **追求**：尽量捕捉 **有价值的情绪拐点**；单日 **少量高质量信号**（经验上每天几个即可）。
- **避免**：信号过多（噪音）或过少（遗漏关键拐点）。

### 11.2 技术目标

1. 在 **`BACKTEST_DATA_SOURCE='tdx'`**（与实盘同源）前提下调参，必要时 **微调逻辑**（非大改架构）。
2. 以 §4 akshare 参考图为 **形态与密度参照**（买卖点是否落在合理拐点），但以 **通达信回测结果** 作为是否上线的验收标准。
3. 控制信号频率：全市场监控时，单票日均信号宜在「几个」量级；需结合 `WAVE_END_REQUIRE_EXCURSION`、冷却与三分门槛联动调节。
4. 评估方式建议：
   - 固定样本：`300959` + 用户自选 3~5 只，区间 `2026-05-22` ~ `2026-05-27`（可扩展）。
   - 对比指标：信号数/日、BUY/SELL 比、与用户标注合理拐点的重合度（主观 + 分时图）。
   - 双轨对照：`tdx` 主结果 vs `akshare` 参考图，避免再次「回测好看、实盘没有」。

### 11.3 已知约束与风险（优化时需牢记）

- 硬门槛（setup/confirm/total）对 **数据源微差** 极敏感；调参时优先看 **通达信** 分数分布，勿仅拟合 akshare。
- `WAVE_END_REQUIRE_EXCURSION=True` 会显著减少信号（同日 8→5 条已验证）；松紧需与信号密度目标一并考虑。
- 过拟合风险：勿针对单票单日把阈值拧到完美；优先 **跨日、跨票** 可接受的稳健区间。

| P2 | 若调参不足，评估温和逻辑微调（如：候选入池规则、边界分数软门槛、早盘/尾盘过滤） |
