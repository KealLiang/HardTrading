请基于Python开发一个「永久投资组合」回测与跟踪工具，满足以下核心需求：

#### 一、核心策略规则
1. 资产配置：永久投资组合由4类资产等权组成（各25%），支持自定义标的（基于本地ETF日线数据）；
2. 再平衡规则：
   - 可指定再平衡频率（月度/季度/年度），均在对应周期的「最后一个交易日」执行；
   - 再平衡逻辑：强制将4类资产仓位拉回各25%（卖出超配资产，买入低配资产）；
   - 不支持主观择时再平衡，仅按固定周期执行。

#### 二、数据源要求
1. 读取本地ETF日线数据（数据在 data/etfs/ 文件夹里，字段包含：日期,开盘,收盘,最高,最低,成交量,成交额,振幅,涨跌幅,涨跌额,换手率）；
2. 无需对接外部数据源，仅使用本地数据。

#### 三、核心功能
##### 功能1：回测功能
1. 输入参数：初始资金、建仓日期、终止日期、再平衡频率（月度/季度/年度）、4类资产标的代码；
2. 回测逻辑：
   - 建仓日按等权（各25%）买入4类资产；
   - 按指定频率在周期最后一个交易日执行再平衡；
   - 计算每日资产持仓市值、总资产、各资产占比、累计收益、年化收益、最大回撤；
3. 输出要求：
   - 以Markdown格式输出详细回测结果，包含：
     - 基础参数汇总（初始资金、时间区间、再平衡频率、标的）；
     - 关键业绩指标（累计收益、年化收益、最大回撤、夏普比率）；
     - 月度/年度收益明细表格；
     - 每一次再平衡的操作记录（日期、各资产卖出/买入金额、调整后仓位）；
     - 每日总资产变动表格（可选展示关键节点）。

##### 功能2：跟踪模式
1. 输入参数：建仓日期、初始资金、当前日期、再平衡频率、4类资产标的代码；
2. 输出要求：
   - 最近一次需执行再平衡的日期；
   - 当前各资产持仓市值、占比；
   - 再平衡操作建议（每类资产需卖出/买入的金额，调整后目标仓位25%）；
   - 以简洁文本/表格形式输出操作指令。

#### 四、代码要求
1. 代码结构清晰，分函数实现（数据读取、回测核心逻辑、再平衡计算、结果输出）；
2. 关键逻辑添加注释，便于理解；
3. 输出结果格式规范，Markdown表格可直接复制使用；
4. 处理日期边界问题（如季度最后一个交易日、年度最后一个交易日的判定）；
5. 计算收益时使用复权价，确保数据准确。

注意：
1. 再平衡时的交易价格使用当日收盘价；
2. 暂时忽略交易成本、手续费、税费（可以作为策略参数，默认值都是0，如果之后要计算带费的收益，修改参数即可）；
3. 若再平衡当日无数据（非交易日），顺延至下一个交易日执行；
4. 跟踪模式需基于当前最新数据，计算「应执行的再平衡操作」，而非历史回测。
